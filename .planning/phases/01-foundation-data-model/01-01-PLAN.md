  <action>
    Create `src/collections/Media.ts` with the full Media collection config:

    1. Upload config:
       - `staticDir: 'media'`
       - `mimeTypes: ['image/*']`
       - `adminThumbnail: 'thumbnail'`
       - Four imageSizes optimized for jewelry photography:
         - thumbnail: 400x400, centre crop, WebP quality 80
         - card: 800x800, centre crop, WebP quality 82
         - detail: 1600 width (no height constraint), WebP quality 85
         - zoom: 2400 width, WebP quality 88

    2. Fields:
       - `alt` (text, required) -- accessible alt text for images
       - `blurDataURL` (text, admin hidden) -- populated by afterChange hook

    3. Access control:
       - read: `() => true` (public)
       - create/update/delete: `isAdmin` (checks `req.user`)

    4. afterChange hook (`generateBlurDataURL`):
       - Skip if not an image (check mimeType)
       - Use Sharp to read the uploaded file from disk: `path.resolve(process.cwd(), 'media', doc.filename)`
       - Resize to 20x20 pixels (fit: 'inside'), convert to WebP quality 20
       - Encode as base64 data URI: `data:image/webp;base64,{buffer}`
       - Update the doc via `req.payload.update()` to set blurDataURL
       - Use a guard to prevent infinite loop (if blurDataURL already set and matches, skip)

    5. Update `payload.config.ts` to import and register Media in the collections array.

    IMPORTANT: The afterChange hook calls `req.payload.update()` which triggers another afterChange. Add a guard: if `doc.blurDataURL` is already populated (from this very update), return early. Check by comparing: if the operation context indicates it's a blur update, skip. Simplest approach: check if `doc.blurDataURL` is truthy and the current update only changed blurDataURL.

    Alternative guard approach: Use a `beforeChange` hook instead of afterChange -- compute blur during the save rather than after. This avoids the recursive update entirely. Choose whichever is cleaner.
